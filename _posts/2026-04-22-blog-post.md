## Summary of ConnectWise Automate Critical API Vulnerability (06/12/2020)

A critical security vulnerability (CVSS 7.8) has been identified in the ConnectWise Automate API, posing a significant risk to MSPs and their clients. The flaw allows a remote authenticated user to execute arbitrary commands or make unauthorized modifications within an Automate instance. This is not a theoretical risk; it is an active, high-priority vulnerability that requires immediate attention due to its potential for system compromise and data breach.

This advisory is highly relevant for all technical operators managing ConnectWise Automate, whether hosted in the cloud or on-premise. The remediation path differs based on deployment model and software version, making precise action critical. Failure to address this promptly could lead to a security incident with direct liability implications for the MSP.

### Key Insights
*   **Severity & Mechanism:** The vulnerability is rated **Critical (Priority 1)** with a CVSS score of 7.8. It enables **authenticated remote command execution** within the Automate environment.
*   **Cloud vs. On-Premise Remediation:** ConnectWise has applied initial mitigations for **cloud-hosted partners**. **On-premise partners** must take manual action based on their specific version.
*   **Version-Specific Patching:** For on-premise installations, a hotfix is required for versions 2020.1 through 2020.5. Partners on 2019.12 or earlier must implement interim mitigation steps immediately, with a formal patch forthcoming.

### Actionable Takeaway
**Immediately verify your Automate version and deployment model.** Cloud partners should confirm with ConnectWise that mitigations are applied to their instance. On-premise partners **must apply the specific hotfix linked for their version (2020.5 - 2020.1) and then re-implement the referenced mitigation steps.** Partners on 2019.12 or older must implement the mitigation steps now and watch for the promised patch.

### Compliance & Security Implications
*   **High-Risk Exploit:** This is a command execution vulnerability. Unpatched systems are actively vulnerable to complete compromise, data exfiltration, and ransomware deployment.
*   **Regulatory Exposure:** Failure to patch could constitute a violation of cybersecurity frameworks (e.g., NIST, CIS Controls) and data protection regulations (like CCPA or GDPR if client data is accessed), leading to significant legal and financial liability.
*   **MSP Liability:** As the managing entity, the MSP bears direct responsibility for applying this patch. An exploit could breach multiple client environments simultaneously, escalating the incident severity.
